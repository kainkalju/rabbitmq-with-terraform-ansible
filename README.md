# RabbitMQ 3-Node Cluster on AWS

Provisions and configures a production-grade 3-node RabbitMQ cluster on AWS `eu-north-1` (Stockholm) using Terraform, Ansible, and Docker.

## Stack

| Layer | Tool |
|---|---|
| Infrastructure | Terraform |
| Configuration | Ansible |
| Runtime | Docker + Docker Compose (managed by systemd) |
| Instances | EC2 t4g.medium — ARM64/Graviton3 |
| OS | Ubuntu 24.04 LTS arm64 |
| RabbitMQ | 4.2.5-management (official multi-arch image) |
| Logging | rsyslog via Unix socket, rotated hourly |

## Architecture

```
eu-north-1 — Default VPC (172.31.0.0/16)
├── eu-north-1a → rabbitmq-1  (seed node)
├── eu-north-1b → rabbitmq-2  (joins rabbitmq-1)
└── eu-north-1c → rabbitmq-3  (joins rabbitmq-1)
```

One node per availability zone. Nodes communicate over private IPs within the VPC. Client access (AMQP 5672, Management UI 15672) is restricted to trusted IPs via existing security groups.

## Prerequisites

- AWS CLI configured with credentials for `eu-north-1`
- Terraform >= 1.6
- Ansible >= 2.14
- Python 3 with `pyyaml` (`pip install pyyaml`)
- SSH key at `~/.ssh/rabbitmq_ed25519` (public key uploaded to AWS as key pair)

## Deployment

### 1. Provision infrastructure

```bash
cd terraform/
terraform init
terraform plan
terraform apply
```

`terraform plan` should show **5 resources**: 1 security group + 1 key pair + 3 EC2 instances.

### 2. Generate Ansible inventory

```bash
terraform output -json | python3 ../scripts/gen_inventory.py > ../ansible/inventory/hosts.yml
```

### 3. Set secrets

`ansible/group_vars/all/vault.yml` is gitignored and must be created locally. Create it with your values:

```yaml
# ansible/group_vars/all/vault.yml
rabbitmq_erlang_cookie: "change-me-to-a-long-random-string"
rabbitmq_user: "admin"
rabbitmq_password: "change-me-to-a-strong-password"
```

Then encrypt it:

```bash
cd ../ansible/
ansible-vault encrypt group_vars/all/vault.yml
```

The vault holds three values:

| Variable | Description |
|---|---|
| `rabbitmq_erlang_cookie` | Shared secret required for Erlang clustering — set once, never change |
| `rabbitmq_user` | RabbitMQ admin username |
| `rabbitmq_password` | RabbitMQ admin password |

### 4. Run the playbook

```bash
ansible-playbook -i inventory/hosts.yml site.yml --ask-vault-pass
```

Playbook runs in order: `docker` → `docker-syslog` → `rabbitmq` (serial, seed first).

### 5. Verify the cluster

```bash
ansible rabbitmq_all -i inventory/hosts.yml -m shell \
  -a "docker exec rabbitmq rabbitmqctl cluster_status"
```

All three nodes should appear under `running_nodes` with no `partitions`.

Management UI is available at `http://<any-node-public-ip>:15672`.

## Security Groups

Each instance attaches:

| SG | Purpose |
|---|---|
| `ssh-enabled` | SSH from trusted IPs |
| `from-home` | All traffic from home/office IPs — covers AMQP and management UI |
| `ping` | ICMP from anywhere |
| `rabbitmq` (new) | Inter-node ports, self-referencing ingress only |

Inter-node ports opened within the `rabbitmq` SG:

| Port | Purpose |
|---|---|
| 4369 | Erlang epmd — node discovery |
| 5672 | AMQP inter-node |
| 25672 | Erlang distribution |
| 35672–35682 | CLI tools |

## Logging

Docker is configured to send all container logs to rsyslog via a Unix datagram socket (`/run/docker/dev-log`). Logs are written to `/var/log/docker/` on each host:

```
/var/log/docker/
├── docker.log       # dockerd and containerd daemon logs
└── rabbitmq.log     # rabbitmq container output
```

Logrotate runs hourly, rotates at 10 MB, keeps 8 compressed generations.

```bash
# Follow live RabbitMQ logs on a node
tail -f /var/log/docker/rabbitmq.log
```

## Re-running the Playbook

The `rabbitmq` role guards the cluster join step — it checks cluster status before running `join_cluster`, so re-running the playbook on an already-clustered node is safe.

## Teardown

```bash
cd terraform/
terraform destroy
```
